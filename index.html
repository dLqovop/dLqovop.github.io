왜 Jekyll가 아닌
Hugo를 선택했는가

Jekyll는 Ruby기반 Static Website Generator로 사용법이 간단하고 근본에다 인기도 많지만, 느린 속도와 템플릿 수정 및 문법의 제약 등의 단점이 있다. 이와 달리 Hugo는 반응 속도가 빠르며, 윈도우를 공식적으로 지원한다. 또한, Jamstack사이트에서 상위 Next.js, Hugo, Gatsby, Jekyll 까지의 테마를 둘러본 결과 맘에 드는 테마가 Hugo에만 있었다.

Hugo의 장점

Hugo는 Golang 기반의 Static Website Generator로 문서화가 잘 되어있으며, 런타임에 의존성이 필요하지 않아 빌드가 빠르다. 단, 별도의 빌드과정이 필요하다. 또한, 윈도우를 공식 지원하고 있어 필자가 사용하기 편리하다.

Static stie와
Static Website Generator

웹사이트는 Static site와 Dynamic stie로 나눌 수 있다. Dynamic site는 상황에 따라 동적으로 html을 생성하여 유저와의 상호작용이 가능하지만, 동적이기 때문에 속도가 느리고 복잡하다. 또한 해당 페이지는 이런 상호작용 보다는 글만 보여주는 단방향성을 띄기 때문에 굳이 Dynamic site로 만들 이유가 없으며 심지어 Github page는 Dynamic site를 지원하지 않는다. 이러한 Static site를 만들어주는 Static Website Generator는 알아서 html을 생성해준다. 보안 관련 문제를 고려할 필요가 없어 작성한 html을 서버에 띄우기만 하면 된다. 또한 Static Website Generator는 다른 사람이 만들어둔 테마를 가져와 쉽게 사용할 수 도 있다.

현재 테마의 라이선스

CC-BY 3.0(Creative Commons Attribution 3.0 License)는 조건 하에 권리를 갖는다.
조건 : 적절한 출처와, 해당 라이센스 링크를 표시하고, 변경이 있는 경우 공지하는 등의 저작자 표시, 라이선스를 제한하는 법적 조건이나 기술적 조치를 부가해서는 안된다.
권리 : 복제 배포, 전시, 공연 및 공중송신 등의 공유가 가능하며 리믹스, 변형, 2차적 저작물의 작성, 영리목적으로도 이용이 가능하다.

  • GPL(GNU-Ceneral Public License)

    자유 소프트웨어 재단(FSS)에서 만든 라이선스로, 수정·배포시 무조건 GPL로 공개하여야 한다. 2차 저작물 또한 반드시 GPL라이선스로 명시한다.

  • AGPL(GNU-Aflero GPL)

    GPL 기반 라이선스로, 웹 서비스를 포함한다. 네트워크로 연결되어 연동되는 프로그램의 전체 소스코드를 공개해야 한다.

  • LGPL(GNU-Lesser GPL)

    GPL 라이선스 사용 시 소스코드 공개의 부담을 보완하기 위해 만들어진 라이선스로, 전체 소스코드를 공개하지 않고 사용된 오픈소스 라이브러리에 대한 소스코드만 공개한다. 한정된 라이브러리에만 적용된다는 뜻으로 Library GPL이라 불렀었지만, 모든 라이브러리에 해당된다는 오해로 Lesser GPL로 명칭이 변경되었다.

  • MIT License

    MIT에서 학생 지원을 위해 만든 라이선스로, 라이선스 저작권 관련 명시만 시키면 되는 느슨한 조건을 가지고 있다.

  • Apache License

    아파치 재단에서 자신들이 관리하는 소프트웨어에 적용하기 위해 자체적으로 만든 라이선스로, 아파치 라이선스 코드를 수정·배포할 시 아파치 라이선스 2.0을 반드시 포함시켜 아파치 재단의 소프트웨어임을 명시하여야 한다.

  • BSD(Berkeley Software Distribution) License

    GPL, LGPL보다 조건이 덜 제한적으로 소스코드를 공개하지 않아도 된다는 큰 이점을 가지고 있다. 소프트웨어 배포 시 저작권 표시 및 보증 책임이 없다는 것을 명시하며, 라이브러리 일부가 수정되어도 소스코드를 공개하지 않아도 된다. 즉, 상용 소프트웨어에서 무제한적으로 사용이 가능하다.
